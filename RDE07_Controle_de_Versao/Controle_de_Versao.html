Controle de Versão

Software de Versionamento

O Software de Versionamento tem como objetivo realizar o controle de versões do código ou de um documento, fazendo com que seja possiível a visualização e gerenciamento de versões dos mesmos realizados por um desenvolvedor ou uma equipe de desenvolvedores

Normalmente, quando vamos escrever um código, trabalho, artigo, etc, montamos uma pasta, criamos o arquivo que desejamos, e caso o arquivo tenha que ser atualizado ou corrigido, nós normalmente criamos arquivos novos para isso. Fazendo com que fique muito confuso.

Os Software de Versionamento fazem oa controles de arquivos através da sua primeira versão. Primeiramente ele cria um repositório com a sua primeira versão, e através dele, ele cria um histórico de versões. Dentro do repositório local, você pode criar um repositório remoto, caso você trabalhe em equipe ou prefira que seus arquivos fiquem na nuvem.

Os Software de Versionamento é de muita importância, pois se já nos confundimos com vários versões de um mesmo arquivo, imagine se tivermos que trabalhar co vários. E també, a casos, que várias pessoas mechem em um mesmo arquivo, com os softwares de versionamento, a equipe consegue ficar a par das atualizações feitas por caad um, e caso apague algo sem querer, não perderá tudo.

Para trabalhar com o seu repositório remoto, você precisará de um servidos específico, que vai gerenciar o repositório local, assim como o repositório remoto, para que os dois estejam iguais.

Por exemplo, se temos uma equipe, onde duas pessoas acessam o mesmo arquivo de um em um repositorio local, cada uma terá suas versões em seus repositorios locais, e conforme o arquivo de cada um for alterado, o software de versionamento vai verificar em cada repositorio, deixando as duas pessoas com as versões atualizadas. Mas caso eles façam o controle por repositorio remoto, o software de versionamento vai controlar as alterações, podendo deixar ou não que mude alguma coisa, e também vai notificando a cada usuários quando uma alteração é feita. Ficando muito mais fácil.

Após o repositorio remoto fazer as alterações, ele cria uma pasta com as versões corrente, ou seja, todas as versões, e as mesmas são destribuidas para os usuarios, deixando-os cientes de cada alteração feita.

Tipos de controle de versão

Centralizado

Possui um repositorio central em um servidor com controles de versões. O problema é que se o servidor parar, não tem como as pessoas trabalharem. Ou seja, é totalmente dependente do servidos.

Exemplo: Software Subversion

Destribuido

Possui um repositorio local com controle de versões. Onde a pessoa pode trabalhar localmente, e quando concluí seu trabalho, ele atualiza no repositorio central. Ele utiliza o servidor, mas é independente do mesmo, ou seja, se o servidor parar eu posso trabalhar localmente, e quando o servidor voltar, a versão é atualizada.

Exemplho: Software Mercurial e Git.

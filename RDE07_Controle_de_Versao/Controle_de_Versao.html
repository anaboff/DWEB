Controle de Versão

Software de Versionamento

O Software de Versionamento tem como objetivo realizar o controle de versões do código ou de um documento, fazendo com que seja possível
a visualização e gerenciamento de versões dos mesmos realizados por um desenvolvedor ou uma equipe de desenvolvedores.

Normalmente, quando vamos escrever um código, trabalho, artigo, etc, montamos uma pasta, criamos o arquivo que desejamos, e caso o arquivo
tenha que ser atualizado ou corrigido, nós normalmente criamos arquivos novos para isso. Fazendo com que fique muito confuso.

Os Softwares de Versionamento fazem os controles de arquivos através da sua primeira versão. Primeiramente ele cria um repositório com a sua
primeira versão, e através dele, ele cria um histórico de versões. Dentro do repositório local , você pode criar um repositório remoto, caso você tra-
balhe em equipe ou prefira que seus arquivos fiquem na nuvem.

Os Softwares de Versionamento é de muita importância, pois se já nos confundimos com várias versões de um mesmo arquivo, imagine se tivermos
que trabalhar com vários. E também, a casos, que várias pessoas mechem em um mesmo arquivo, com os softwares de versionamento, a equipe con-
segue ficar a par das atualizações feitas por cada um, e caso apague algo sem querer, não perderá tudo.

Para trabalhar com o seu repositório remoto, você precisará de um servidor específico, que vai gerenciar o repositório local, assim como o repositório
remoto, para que os dois estejam iguais.

Por exemplo, se temos uma equipe, onde duas pessoas acessam o mesmo arquivo de um em um repositório local, cada uma terá suas versões em seus
repositórios locais, e conforme o arquivo de cada um for alterado, o software de versionamento vai verificar em cada repositório, deixando as duas pes-
soas com as versões atualizadas. Mas caso eles façam o controle por repositório remoto, o software de versionamento vai controlar as alterações, poden-
do deixar ou não que mude alguma coisa, e também vai notificando a cada usuários quando uma alteração é feita. Ficando muito mais fácil.

Após o repositório remoto fazer as alterações, ele cria uma pasta com as versões corrente, ou seja, todas as versões, e as mesmas são distribuídas para os
usuários, deixando-os cientes de cada alteração feita.

Tipos de controle de versão

Centralizado

Possui um repositório central em um servidor com controles de versões. O problema é que se o servidor parar, não tem como as pessoas trabalharem.
Ou seja, é totalmente dependente do servidos.

Exemplo: Software Subversion

Distribuído

Possui um repositório local com controle de versões. Onde a pessoa pode trabalhar localmente, e quando concluí seu trabalho, ele atualiza no repositório
central. Ele utiliza o servidor, mas é independente do mesmo, ou seja, se o servidor parar eu posso trabalhar localmente, e quando o servidor voltar, a versão
é atualizada.

Exemplo: Software Mercurial e Git.

Repositórios:

Repositório Remoto:

Repositórios Remotos são repositórios presentes em outra máquina para a qual nós podemos pegar ou enviar código.

Repositório Local:

Repositório Local é um repositório presentes em sua máquina onde podemos pegar ou salvar códigos.

Preparando o Ambiente

Para preparar o ambiente você vai precisar dos seguintes aplicativos:

  • GitHub – www.github.com/
  • Git – https://git-scm.com/
  • GitHub Desktop - https://github.desktop.com/
  • VScode - https://code.visualstudio.com

E então você vai seguir os seguintes passos:

  1. Baixar o Git
  2. Criar/logar uma conta no GitHub
  3. Baixar o GitHub Desktop
  4. Baixar o VScode

Comandos Git

Alguns comandos que serão úteis, esses comandos não serão usados em linhas de comandos, pois estamos utilizando o GitHub desktop, mas ouviremos/usaremos
muito os mesmos.

commit: torna permanente um conjunto de alterações repositório;

push: envia alterações para o servidor repositório;

fetch: verifica se o seu repositório local está em sincronia com o remoto;

pull: sincroniza o repositório local com o remoto atualizando as versões.

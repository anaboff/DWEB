Desenvolvimento Web

Como se deu a evolução da internet

A internet surgiu a partir da guerra fria entre EUA x URSS (em 1947), pois os dois países queriam mostrar qual deles era o mais poderoso.
Em 1957 o URSS lançou Sputnik na órbita da terra. Então os EUA acharam que era uma estratégia deespionagem e em 1958 criaram o
DARPA (Denfense Advanced Research Agency).

Em 1969 teve a criação da ARPANET (Protocolo NCP) que conectava 4 computadores na Costa Leste, entre institutos de pesquisa e institu-
tos militares. Em 1970 a ARPANET teve sua criação para a Costa Leste. Como o protocolo NCP estava difícil de lida, chamaram então os
pesquisadores Robert Khan e Vint Cerf, e criaram o protocolo TCP em 1973, que ficou em desenvolvimento até a década de 80. Com o au-
mento da rede eles fizeram uma divisão entre rede militar, científica e comercial.

Em 1963 o Milnet NSF Comercial criou o protocolo de TCP/IP. O TCP é um protocolo de transferência de arquivos, e antes não tinha o IP,
ou seja, não tinham identificação. O IP serve como identificação de cada máquina. No ano de 1989 tivemos o primeiro domínio “.br”.

No ano de 1990 Tim Berners-Lee cria o World Wide Web(WWW), também foi criado o protocolo http , linguagem HTML, e surgiu o proto-
colo Gopher.

Em 1994 Tim Berners-Lee criou o W3C que padroniza o WWW, e neste mesmo ano foi criado o URL. No ano de 1955 foi criada a lingua-
gem JavaScript e CSS que é de esstilização. Neste mesmo ano foram criados os primeiros e-commerces Amazon, Ebay. No ano de 1997 foi
registrado o domínio Google, que começou a funcionar em 1998. Em 2001 começou o estudo sobre a “Internet das coisas”
como internet mobile,apps,smarts tv. No ano de 2004 foi criado o facebook e em 2005 o Youtube.

Principais invenções que proporcionaram o desenvolvimento da internet

As principais invensões que ajudaram no desenvolvimento da internet foram o protocolo TCP, que depois virou o TCP/IP, o IP possibilitando
a identificação das máquinas. Também a criação do HTTP,HTML,WWW,W3C.

Protocolos http e https

Os protocolos HTTP e HTTPs são servidores específicos para hipertextos, ele é um sistema que permite visualizar as informações de documen-
tos que possuem ligações e que permitem chegar a outros documentos.

Função do DNS

O DNS é um Sistema de Nomes de Domínio, que permite a busca de sites sem saber o IP, pois na hora que você digita a URL do site o seu na-
vegador acessa o servidor DNS e verifica qual o IP do site, retornando o IP para seu servidor, permitindo o acesso ao site. A existência do DNS
é muito importante porque nos permite fazer buscas em vários sites, sem ter que saber qual o IP do servidor. Sem ele teríamos que ter uma lista
gigante de IP’s e ficaria muito mais difícil a utilização da internet.

Protocolo TCP/IP

O protocolo TCP/IP é o modo com que a máquina acesse os dados do servidor, fazendo as transferências em pequenos pacotes de dados, transmi-
tindo até chegar à máquina.

Hipertexto

Hipertexto é um sistema que permite a visualização de informações de documentos que contém links e hiperlinks, e que nos permitem chegar a
outros documentos.

Projeto W3

O Projeto w3 tem o propósito de fazer com que os artigos e dados encontrados na internet sejam acessados por todos sem ser dominado ou altera-
do por um servidor de hospedagem. Todos terão acesso a internet, e todo os dados contidos nela ficaram na nuvem. Os usuários poderão escolher
onde armazenar seus artigos e dados, e também acessar sem ter que ver anuncio.

A W3 pretende um futuro para internet utilizando a inteligência artificial. As máquinas pensarão como os humanos. Não vamos mais precisar digi-
tar o link do site desejado, só precisará clicar no mouse ou teclado e digitar a palavra chave desejada. O futuro da internet mudará completamente,
pois a utilização da IA será muito importante para esse processo.

Tecnologias

HTML

É uma linguagem que se baseia em tags e funciona como marcação de hipertextos. Foi criada em 1990 por Tim Berness-Lee para poder publicar
conteúdos de hipertexto como:

  • texto;
  • imagem;
  • áudio;
  • vídeo.
E poderem ser vinculados através de hiperlinks.

A HTML 5 é utilizado para é uma linguagem de marcação de texto e responsável pelo conteúdo do site como texto, vídeo, imagem e tabela. No
HTML 5 nós utilizamos tags. As tags definem ao navegador a marcação de conteúdo. As tags tem funcionalidades específicas. Sem o uso de tags
o navegador exibe um texto plano. As tags podem possuir atributos, ou seja,são parâmetros que definem algum tipo de informação extra ao nave-
gador.

CSS

O CSS foi proposto em 1994, por Hakon Lie, com a intenção de facilitar a programação de sites que era muito complexa na época. Em 1995 a W3C
desenvolveu o CSS 1, e a linguagem ficou muito popular entre os anos de 1997 e 1999

O CSS 3 é utilizado para a estilização de conteúdo, como cores, fontes, tamanho, sombras, posicionamento. A linguagem é utilizada em folhas de es-
tilo cascata, ela tem uma sintaxe simples, com palavras em inglês, que especificam os nomes de algumas propriedades de estilo de uma página.

Java

O JavaScript foi criado por Brendan Eich na década de 90, a serviço da empresa Netscape. Ela foi criada para fazer com que a web se tornasse mais di-
nâmica. No ano de 1995 a linguagem foi adicionada ao Netscape Navigator.

O JavaScript é uma linguagem utilizada para a interatividade com o usuário, como menus interativos, popups, validação e animação. Vale lembrar que
o JavaScript não tem relação com a linguagem Java. Ela é uma linguagem extremamente flexível e pode ser utilizada tanto no frontend como no backend.

Como funcionam as tecnologias HTML 5, CSS 3 e JavaScript

Depois que o usuário digita o link na área de navegação e aperta a tecla enter é mandada uma requisição para o servidor DNS, onde ele vai buscar o IP
da página, e assim retorna o IP para a máquina do cliente, que dispara uma requisição a página digitada, que disponibiliza as páginas HTML,CSS,
JavaScript para o navegador, que lê as mesmas.

Página Web e Website

A página Web ou Webpage é um arquivo onde você pode inserir tabelas, textos, imagens e vídeos, ou incorporar planilhas em uma página com ligações
de hipertexto que levam a outras seções, para outras páginas

Websites ou sites

Websites ou sites são uma coleção de páginas web, que podem ser armazenadas e uma pasta ou subpastas relacionadas e um servidor web

Arquitetura Cliente-Servidor

Como vimos, a internet funciona em uma arquitetura servidor, como mostrado na parte ”Como funcionam as tecnologias HTML 5, CSS 3 e JavaScript”.
O servidor ou como é chamado de Server-Side deve ter tecnologias que atendam ao usuário, que podem ser linguagem de programação, banco de dados,
que são:

  • Java
  • PHP
  • MySQL
  • Python
  • Ruby

Também é necessário, que a parte do usuário ou como é chamado de Client-Side, tenha tecnologias para ler as páginas que foram enviadas do Servidos, as
mais usadas são HTML 5, CSS 3 e JavaScript que explicamos na parte “Tecnologias”, além de outras. Outra estrutura extremamente necessária é que te-
nham profissionais que trabalhem no lado do Server-Side.

Profissionais

Os profissionais que trabalham nessa área de Desenvolvimento Web são:

Front-End

Os profissionais Front-End trabalham na área do Cliente-Side, com as linguagens HTML, CSS e JavaScript

Back-End

Os profissionais Back-End trabalham na área Server-Side, com as linguagens Java, SQL, PHP e Python.

FullStack

Os profissionais FullStack trabalham tanto com Front-End, quanto com Back-End, e utilizam as linguagens HTML, CSS, JavaScript, Java, SQL, PHP e
Python. Para trabalhar como FullStack o profissional deve ter um grande conhecimento das linguagens de programação, banco de dados, e de tecnologia.
